# Opensource Team 6

## 프로젝트 소개

이 프로젝트는 6조 팀원들의 포트폴리오를 모은 웹사이트입니다.

Opensource Team 6 저장소는 팀원들의 자기소개 페이지를 포함한 웹 프로젝트를 관리하기 위한 공간입니다. 각 팀원은 자신의 HTML 소개 페이지를 작성하고, 팀 전체가 협업하여 하나의 프로젝트를 구성합니다.

메인 배너에 팀원 네명의 프로필과 노래가 있습니다. 프로필에 커서를 가져다대면 노래 재생과 함께 프로필 사진이 앨범 커버로 바뀝니다. 한 팀원의 프로필 칸을 누르면 해당 팀원이 고른 노래와 함께 팀원에 대해 알아볼 수 있습니다! 
* 6조의 취향이 담긴 개인 공간으로 놀러오세요!!
---

## 파일 구성

* **aejin.html**: 애진 소개 페이지
* **changmin.html**: 창민 소개 페이지
* **jin.html**: 진근 소개 페이지
* **yerim.html**: 예림 소개 페이지
* **index.html**: 팀 메인 페이지

---

## 주요 기능

* 각 팀원 개별 소개 페이지 제공
* 메인 페이지에서 팀원 소개 페이지로 이동 가능
* GitHub 기반 협업 및 버전 관리

---

## Git 사용 규칙

1. main branch를 생성해 로컬에서 각자 작업 후 원격에 push
2. main에 push하기 전 반드시 최신 버전 pull
3. 충돌 발생 시 팀원과 협의하여 해결
4. 커밋 메시지는 명확하게 작성
5. 배포 전 release branch 제작해서 오류 수정 및 페이지 발전
6. release에 push하기 전 최신 버전 pull
7. 커밋 메세지는 명확하게 작성

## 프로젝트 실행 방법 (로컬 환경)
내 컴퓨터에서 직접 코드를 다운로드하여 실행하려면 아래 단계를 따라주세요.

1. 저장소 클론
터미널을 열고 아래 명령어를 입력하여 프로젝트를 다운로드합니다.

git clone [https://github.com/aejin-cloud/Opensource_team-6.git]

3. 프로젝트 폴더로 이동
터미널에 아래 명령어를 입력합니다.

cd Opensource_team-6

3. 실행
폴더 내의 index.html 파일을 더블 클릭하거나 브라우저로 열어서 확인합니다.

---

## 협업 방법

* GitHub Issues를 활용한 작업 관리
* Main branch를 만들고, 개인 작업 이후 git add, git commit, git pull origin main, git push origin main을 통해 원격 저장소에 push
* Release branch를 만들어 배포 전 수정사항 고치기
* 완성이 되면 release branch를 main branch와 합치기
